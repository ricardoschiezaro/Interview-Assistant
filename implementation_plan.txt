INTERVIEW ASSISTANT - TECHNICAL IMPLEMENTATION PLAN & CURRENT STATE

1. PROJECT OVERVIEW
The Interview Assistant is a low-latency, high-availability desktop application (PyQt6) designed to provide real-time AI-generated responses during interviews. It captures audio (Mic or System), transcribes it via Deepgram, sanitizes the text, and queries a chain of LLM providers with automatic fallback logic.

2. CORE ARCHITECTURE (THE PIPELINE)
The application uses an asynchronous producer-consumer pattern with asyncio.Queue:

- Audio Capture (core/audio_capture.py):
  * Uses a dedicated thread to capture PCM audio.
  * Supports Hybrid Audio: Switching between Microphone and System Audio (Loopback).

- STT Engine (pipeline/stt.py):
  * Powered by Deepgram (Streaming WebSocket).
  * Optimized for sub-second transcription latency.

- Sanitization (pipeline/sanitizer.py):
  * Optional step using Cloudflare Workers/AI to clean up raw transcripts (fixing typos, punctuation, and context).

- LLM Responder (pipeline/responder.py):
  * Circuit Breaker Fallback System: If one provider fails or times out (0.8s - 2.0s), it immediately tries the next.
  * Provider Order:
    1. Cerebras (Llama 3.1 70b) - Ultra-fast.
    2. Groq (Llama 3.3 70b).
    3. Groq (Llama 3.1 8b).
    4. Google Gemini 2.0 Flash (Final fallback).

- Orchestrator (pipeline/orchestrator.py):
  * The "glue" that connects the queues and manages the lifecycle of all components.

3. UI IMPLEMENTATION (PYQT6)
- Design: Modern Dark Mode / Glassmorphism aesthetic.
- Main Features:
  * Copilot Tab: Real-time streaming display of transcripts and AI answers.
  * Profile Tab: Management of user professional data (Resume/Context) stored in profile.json.
  * Prompt Tab: Configuration of the System Prompt and interview rules.
  * Scratchpad: A quick-note area for the user.
- Signal System: Uses ui/signals.py for thread-safe communication between the background asyncio loop and the PyQt UI.

4. CURRENT FILE STRUCTURE
- main.py: Entry point, initializes the PyQt app and background loop.
- core/: Core logic (Audio capture, Config, Profile management).
- pipeline/: The processing stages (STT, Sanitizer, Responder, Orchestrator).
- ui/: All PyQt window and tab components.
- profile.json: Stores user context and experience.
- .env: API Keys for Deepgram, Groq, Cerebras, Google, and Cloudflare.

5. RECENT ENHANCEMENTS
- Stability: Added robust error handling in the STT WebSocket connection.
- Latency: Implemented sub-second timeouts for LLM providers to ensure responses never hang.
- Hybrid Audio: UI toggle to switch between capturing the interviewer (System Audio) and the user (Mic).
- Clean History: Added ability to reset conversation context.

6. GUIDELINES FOR NEW FEATURES
- Async First: All pipeline logic must remain non-blocking using asyncio.
- Thread Safety: Any updates to the UI from the pipeline must use Qt Signals.
- Failover: New LLM features should respect the existing fallback chain.
